# Auto-Commit Skill

Auto-generated by opencode-meta — 2026-04-21T00:00:00Z

## What I do

- make auto commits after every user prompt and agent invocation
- use the commit message format below

## When to use me

- at the end of every actions right before the user input is required
- at the end of each subagent invocation


## Commit Message Format

```
temp: <agent-name> <summary-max-6-words>

<user-prompt>
```

### Examples

```
temp: backend Add customer API endpoint

Add a new REST endpoint to retrieve customer details by ID
```

```
temp: frontend Implement dark mode toggle

Add dark mode toggle to the application settings with theme persistence
```

## Workflow

1. **After each user prompt is processed**: Check if there are any uncommitted changes using `git status`.
2. **Prepare commit message**:
   - Extract the agent name from the current agent context
   - Create a concise summary (max 6 words) of the changes made
   - Include the original user prompt as the commit body
3. **Stage changes**: Use `git add .` to stage all changes
4. **Create commit**: Use `git commit -m "temp: <agent-name> <summary>" -m "<user-prompt>"` to create the commit
5. **Verify**: Run `git log -1 --oneline` to confirm the commit was created

## Implementation Notes

- Only commit if there are actual changes (check `git status` first)
- Use the agent's display name or identifier for `<agent-name>`
- Keep the summary concise and action-oriented (e.g., "Add", "Fix", "Update", "Refactor")
- The user prompt should be included as-is in the commit body for full context
- Do not force push or rewrite history
- If a commit fails, report the error but continue with the task

## Git Configuration

Ensure the following git configuration is set before committing:

```bash
git config user.name "OpenCode Agent"
git config user.email "agent@opencode.local"
```

If not set, agents should configure these values before attempting commits.

## Error Handling

- If `git status` shows no changes, skip the commit
- If `git add` fails, report the error and continue
- If `git commit` fails (e.g., due to pre-commit hooks), report the error and suggest fixes
- Never force-push or use `--amend` without explicit user instruction

## Integration with Agent Workflow

This skill should be invoked at the end of each agent's task execution, after all changes have been made but before returning control to the user. The agent should:

1. Complete the requested task
2. Verify all changes are working correctly
3. Invoke this skill to commit changes
4. Report the commit hash and message to the user
5. Return control to the user

## Disabling Auto-Commit

If an agent needs to skip auto-commit for a specific task, it should:
- Explicitly state the reason in its response
- Ask the user for confirmation before skipping
- Document why the commit was skipped
